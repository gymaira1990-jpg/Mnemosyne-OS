## 一、产品定位

> **Mnemosyne 是纯个人专业服务**——辅助个人研究、专业工作、对话记忆。不是通用产品，不跟商业SaaS对标。私有部署、自有数据、本地推理。

### 设计哲学

```
1. 个人优先 — 专为一个人的研究和工作流设计，不为多租户优化
2. 数据主权 — 全部数据在自有 PostgreSQL 中，零第三方 API
3. 本地推理 — GPU 在本地 WSL，Embedding/Rerank 也本地
4. 层级记忆 — TMT 五级记忆树（碎片→会话→日报→周报→画像）
5. 热度即是时序 — heat_score 自动管理记忆生命周期
6. 崩溃安全 — SQLite 持久队列 + systemd 三层自愈
```

## 三、核心功能

### 热度体系 (时序机制)

heat_score [0, 1] 自动管理记忆生命周期：

```
存储时: heat_score = 0.5 (新记忆)
衰减: 7天-0.02 → 30天-0.04 → 90天-0.08
加权: 5次+访问且7天内 → +0.05 (上限1.0)
矛盾: invalid_at 非空 → -0.1
层级: >0.7→L1 / 0.2-0.7→L2 / <0.2+30天→L3 / <0.05+90天→L4(删除)
触发: POST /api/v1/reflect (cron 每4h light, 每天4am deep)
```

### TMT 五级记忆树 (蒸馏管道)

```
L1 碎片 (memories)       ← 每条对话/笔记         [338条]
  ↓ 自动蒸馏 (cron */10)
L2 会话 (tmt_sessions)   ← 一轮对话摘要           [16条, 6/7修复]
  ↓ 自动蒸馏 (cron 23:50)
L3 每日 (tmt_daily)      ← 跨会话主题提炼         [2条, 6/7修复]
  ↓ 自动蒸馏 (每周/月)
L4 每周 (tmt_weekly)     ← 模式/趋势综合          [0条, 等数据]
  ↓ 自动蒸馏 (每月)
L5 画像 (tmt_profiles)   ← 人设/偏好固化          [0条, 等数据]
```

每个层级蒸馏使用 WSL 本地 LLM (Qwen3.5-4B GPU, 通过 -R 11435 反向隧道访问), 保底走 GZ CPU Qwen3.5-2B。

### 搜索 (四维混合)

搜索公式 (2026-06-07 修正):

```
score = 0.40 × 语义向量 (pgvector cosine)
      + 0.15 × BM25 关键词 (ILIKE)
      + 0.15 × 时序新鲜度 (7天/30天梯度)
      + 0.15 × 可信度 reliability     ← 修正前仅贡献2.25%
      + 0.15 × 热度 heat_score       ← 修正前仅贡献1.5%
```

### 辨证推理 (Dialectic)

多轮搜索 + L2 会话上下文回传。注意：名为「辨证」但实际是增强搜索，目前不做正反合分析。

### 矛盾检测

写入时自动检测语义相似记忆：
- `dist < 0.15 + ratio > 0.85` → 合并（增访问计数）
- `dist < 0.12 + ratio < 0.5` → 矛盾（旧标记过期，新标记冲突）
- 查询：`GET /api/v1/memories/conflicts`

### Wiki 知识库

版本化文档知识库：`wiki_pages(title, content, version)` + `wiki_versions` 历史追踪

### 信念系统 (Beliefs)

`confidence [0, 1]` + `status (tentative/established/contradicted)` + 自动演化
