## The Problem

Every AI agent today suffers from the same amnesia: conversations reset. Context windows overflow. Important decisions vanish into scrollback. We duct-tape solutions — vector DBs for retrieval, RAG for injection, prompt stuffing for continuity — but none of them **understand** memory. They store bytes. They don't know what matters and what doesn't.

**Mnemosyne OS** is a different answer. It treats memory as a first-class system with its own lifecycle: capture, distill, age, forget, resurface. It runs 7×24 on your own server. It's built for agents that need to remember across weeks, not just within a single `context_length`.

---

## How It Works

Every time you talk to your AI agent, Mnemosyne intercepts the conversation at session end:

```
Raw chat messages
     │
     ▼ conversation_messages (PG, permanent)
     │
     ▼ L1: Memory fragments (extracted, embedded 1024d)
     │
     ├── L2: Session summaries (LLM-distilled, key decisions)
     │
     ├── L3: Daily reports (cross-session themes)
     │
     ├── L4: Weekly patterns (recurring topics, progress arcs)
     │
     └── L5: User profile (stable preferences, identity)
```

Each level is **auto-generated by LLM distillation** — not templated, not heuristic. The same pipeline that processes your chat also handles agent delegation events, memory writes, and context compression hooks.

## What Sets It Apart

### 🧠 Five-Dimensional Search

Not just cosine similarity. Every query scores against five axes:

```
score = 0.40 × semantic (1024d HNSW pgvector)
      + 0.15 × BM25   (full-text keyword)
      + 0.15 × temporal (7d=0.15, 30d=0.08, 90d=0)
      + 0.15 × trust   (reliability score from conflict detection)
      + 0.15 × heat    (access frequency × decay curve)
```

**Time-ordered mode** (`sort=created_at`) skips the hybrid formula entirely — pure chronological retrieval for "what happened recently?" queries. This separates the **time axis** (recency) from the **heat axis** (importance).

```bash
# Hybrid: best for semantic understanding
curl -X POST :8010/api/v1/memories/search \
  -d '{"user_id":"default","query":"PostgreSQL optimization","top_k":5}'

# Time-ordered: best for "what did we do yesterday?"
curl "http://:8010/api/v1/memories?user_id=default&sort=created_at&limit=20&search=deploy"
```

### 🏛️ Three-Hall Knowledge Lifecycle

Knowledge isn't static. It matures:

```
Research Hall    → "we're trying this"
  │ (validated)
Engineering Hall → "this works, here are the pitfalls"
  │ (battle-tested)
Archive Hall     → "this is canonical truth"
```

Memories carry `valid_from` / `valid_to` timestamps. Conflict detection automatically marks superseded knowledge as expired. Nothing rots — it ages gracefully.

### 🔗 Entity Knowledge Graph

Entities (projects, people, concepts, tools) are extracted via LLM + regex and stored as an Apache AGE graph. Not isolated cards — a connected web:

```cypher
MATCH (m:Memory)-[:MENTIONS]->(e:Entity {name: "pgvector"})
RETURN m.content, e.name
```

Multi-hop traversal means asking "what else was discussed alongside the PostgreSQL migration?" returns context, not just keyword matches.

### 💬 Permanent Conversation History

Hermes `state.db` (SQLite) syncs to PostgreSQL `conversation_messages` on every session end. The full exchange — user, assistant, tool calls, reasoning — is preserved with timestamps.

```bash
# List all sessions
curl "http://:8010/api/v1/sessions?limit=20"

# Load a conversation like a chat app
curl "http://:8010/api/v1/sessions/{session_id}/messages?limit=200"

# Pagination support
curl "http://:8010/api/v1/sessions/{session_id}/messages?before_id=500"
```

Crash? `/new` too early? The session is already in PostgreSQL. Your frontend loads it like WeChat loads yesterday's chat.

### 🔌 Agent-Native Integration

**MCP Bridge** — 15 tools exposed directly to your Hermes Agent:

```
mnemosyne_search       — 5D hybrid retrieval
mnemosyne_recall       — Deep dialectic with L2/L3 context
mnemosyne_hot_memories — Heat-ranked trending topics
mnemosyne_remember     — Store with auto-conflict detection
session_search         — Time-ordered session browser
mnemosyne_dialectic    — Multi-hop graph + semantic
mnemosyne_wiki         — Versioned knowledge base
mnemosyne_media        — Multimodal memory (file/image/link)
```

**Memory Provider** — 10 lifecycle hooks that fire automatically:

```
on_session_end    → sync messages + trigger L2 distillation
on_turn_start     → prefetch relevant memories before each turn
on_pre_compress   → inject key context before compression
on_delegation     → log subagent tasks + results
on_memory_write   → mirror Hermes memory writes to Mnemosyne
on_session_switch → flush write queue on session change
```

No manual `remember()` calls needed. The system remembers because it's watching.

### ☁️ Edge-Cloud Resilience

WSL laptop goes offline? Messages cache to local SQLite. Back online? Silent push to GZ PostgreSQL. Seven cron jobs maintain the system: heat decay, duplicate cleanup, entity extraction, TMT distillation, chunk reslicing, offline sync.

## Quick Start

### Prerequisites
- Python 3.12+
- PostgreSQL 16 with pgvector extension
- Apache AGE extension (for knowledge graph)
- 8GB+ RAM (embeddings are 1024-dimensional)

### Hermes Agent (one command)

```bash
hermes mcp add mnemosyne --command python3 \
  --args integrations/hermes-mcp/mnemosyne_mcp.py
```

The MCP tools appear in your agent's toolset immediately after `/reset`.

### Standalone API

```bash
git clone https://github.com/gymaira1990-jpg/Mnemosyne-OS.git
cd Mnemosyne-OS
pip install -r requirements.txt

# Set your embedding/LLM backend
export EMBED_MODEL=doubao-embedding-vision
export ARK_API_KEY=your_key
export LLM_MODEL_LITE=doubao-seed-2-0-lite

python main.py
# → Mnemosyne OS v5.3.2 on :8010
```

### Python SDK

```python
from integrations.sdk import MnemosyneHermesMemory
m = MnemosyneHermesMemory(endpoint="http://127.0.0.1:18010")

m.add("pgvector HNSW outperforms IVFFlat for high-dimensional recall")
results = m.get_relevant("which pgvector index is better?")
# Returns top-5 with scores broken down by dimension
```

# 🇨🇳 中文版

## 🏛️ Mnemosyne OS — 认知型记忆操作系统

**给 AI Agent 用的长期记忆系统。不是向量数据库，不是 RAG 管道。是会自己整理、提炼、老化的记忆 OS。**

### 它解决了什么

每个 AI Agent 都健忘：会话一关全忘光，上下文窗口溢出，重要决策消失在滚动条里。现有方案（向量数据库 + RAG + prompt 拼接）都是打补丁——能存数据，但不懂"什么重要、什么该忘"。

Mnemosyne OS 把记忆当作一等公民：**捕获 → 蒸馏 → 老化 → 遗忘 → 浮现**。你的 Agent 记的不是碎片，是有生命周期的知识。

### 核心能力

**五维搜索** — 不是简单的余弦相似度。每次查询同时评估：语义向量(40%) + 全文关键词(15%) + 时间新鲜度(15%) + 可信度(15%) + 热度(15%)。也支持纯时间排序（`sort=created_at`），热度轴和时间轴分开不混淆。

**五层蒸馏** — 对话结束自动触发：原始碎片(L1) → 会话摘要(L2) → 日报(L3) → 周报(L4) → 用户画像(L5)。每层都是 LLM 自动生成的，不是模板填空。

**三馆闭环** — 知识有生命周期：研究馆（待验证）→ 工程馆（踩过的坑）→ 档案馆（已沉淀的真理）。矛盾检测自动标记过时记忆。

**知识图谱** — 实体和关系构成 Apache AGE 图。不是孤立的记忆卡片，而是一张可多跳遍历的网。

**会话永久记忆** — Hermes 每次会话结束，所有消息（user/assistant/tool/reasoning）同步到 PostgreSQL `conversation_messages`。前端可以像微信一样加载历史对话。意外关闭也不丢。

**Agent 原生集成** — 15 工具的 MCP Bridge + 10 个生命周期 Hook。不需要手动调 `remember()`——系统自己在看、在记、在想。

**端云双活** — WSL 断网？消息缓存到本地 SQLite。恢复后静默推回云端 PostgreSQL。7 条 cron 任务保证一切正常运行。
