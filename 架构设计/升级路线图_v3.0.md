## Phase 1 · 根基加固 ✅ (2026-06-03 完成)

### P1-T1: 持久写队列 (crash-safe) ✅
**参考**: RetainDB `_WriteQueue` 模式 | **投入**: ~210 行
- SQLite 持久队列：sync_turn 先写队列再发，crash-safe
- WSL 重启自动回放待发送条目
- **文件**: `write_queue.py`

### P1-T2: 熔断器 (Circuit Breaker) ✅
**参考**: Mem0 | **投入**: 内置于 T1
- 连续 5 次失败 → OPEN（暂停 120s）→ HALF_OPEN → CLOSE
- 防止 SSH 隧道故障时疯狂重试

### P1-T3: 信任评分 + 反馈 ✅
**参考**: Holographic | **投入**: ~100 行
- search 五维修排序: `0.40*vec + 0.15*BM25 + 0.15*时间 + 0.15*reliability + 0.15*heat`
- feedback 端点: positive(+0.1), negative(-0.1)，metadata 自动计 helpful_count/feedback_count
- **文件**: GZ `main.py`（search + feedback 端点）

### P1-T4: 消息清理 + 临时检测 ✅
**参考**: Supermemory | **投入**: ~80 行
- 自动剥离 `## Mnemosyne 关联记忆` 等上下文注入标签
- 过滤无意义消息（<5字 / 纯符号 / emoji / 单字重复）
- **文件**: `message_cleaner.py`

### Provider 重构
- `mnemosyne_provider.py` — sync_turn 队列模式 + 消息清理（577 行）
- 已部署至 `plugins/memory/mnemosyne/`（下次会话自动生效）
- 新增模块：`write_queue.py`, `message_cleaner.py`
