## Phase 3.5 · 执行断层修复 ✅ (2026-06-07)

> 代码写了，管道没通。审计发现三个重大执行断层，一次修通。

### 问题与修复

| # | 发现 | 修复方式 | 影响 |
|:--|:-----|:---------|:-----|
| **🔴 1** | `tmt.py` 的 APIRouter 从未挂载到 `main.py` | `include_router(tmt_router)` + startup注入全局变量 | TMT L2-L5 蒸馏管道通了 |
| **🔴 2** | GZ 无法调用 WSL GPU LLM（缺 -R 11435 隧道） | autossh 加入 -R 11435, 更新 systemd service | 蒸馏从 CPU(Qwen3.5-2B, 90s+) 转为 GPU(Qwen3.5-4B, ~20s) |
| **🟡 3** | 搜索权重中 `reliability*0.15` 和 `heat*0.10` 导致贡献仅 2% | 改为直接参与排序 | 热度/可信度真正影响排序 |

## v4.0 方向 (基于 2026-06-07 深度调研)

### 核心决策

> Mnemosyne 是 **纯个人专业服务**，不做通用产品。我们的 TMT 层级 + 辨证推理 + 矛盾检测 + 热度体系 是竞品没有的护城河。

### 不做的

- ❌ 不引入 Neo4j/FalkorDB（PG+pgvector+AGE 够用）
- ❌ 不转型全栈 Agent Runtime（Letta 路线）
- ❌ 不依赖商业 SaaS（Supermemory/Zep Cloud）
- ❌ 不从零手撸可视化（用 vis-network 快速 MVP）

### 待推进方向

| 优先级 | 方向 | 说明 |
|:------|:-----|:------|
| P0 | 实体解析 Pipeline | 当前是纯正则，需要 spaCy→GLiNER→LLM 三级级联 |
| P0 | 可视化 MVP | React + vis-network 图谱视图 |
| P1 | 信念系统增强 | Fact vs Belief 分离，参考 Hindsight 四网络 |
| P2 | 评测基准 | LoCoMo/LongMemEval |
